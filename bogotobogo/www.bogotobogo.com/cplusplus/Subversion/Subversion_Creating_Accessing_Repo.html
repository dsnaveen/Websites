Creating a Test Repository

Let's create a repository. We'll see how to create a simple test repository containing one text file, and how to check out and commit files.

The repository will be a subdirectory in the repos directory, and will have its group ownership set to svn.

$ ls -la
drwxrwsr-x  2 root svn  4096 Oct 18 08:56 repos

We also need to make sure the repository will be group writable, so that the other members of the repos group will be able to commit files:

$ umask 002

This sets the new file mode creation mask which controls the default permissions of any new file that we create. The default value is 022 and it corresponds to read/write permissions for the file owner, and read permissions for the group and others. The new value, 002 gives write permissions to the group.

Now, let's create the repository using the svnadmin command:

$ sudo svnadmin create /usr/local/svn/repos/test

And set back the default umask:

$ umask 022

Now we have an empty repository, waiting for us to commit something to it. But, before we do this, we need to check out the current version (i.e., the empty directory) to create a working copy:

$ svn checkout file:///usr/local/svn/repos/test
Checked out revision 0.

The working copy has been checked out to a new directory, test. Let's create a simple "hello world" text file in that directory:

$ cd test
$ echo 'Hello, World!' > hello.txt

Then, add it to version control with the svn add command:

$ svn add hello.txt
A         hello.txt

Finally, commit it using svn commit:

$ svn commit -m "Added a 'hello world'file."
Adding         hello.txt
Transmitting file data .
Committed revision 1.

The hello.txt file is now in the repository.

In that case, we can do the following:

$ pwd
/usr/local/svn/repos
$ sudo chmod -R g+w test
